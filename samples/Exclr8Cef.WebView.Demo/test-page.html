Renderer crash recovery

chrome://crash only works in CEF's Chrome runtime, which conflicts with OSR — so there's no in-page trigger here. Real renderer terminations (OOM, segfaults, killed Helper processes) fire CefBrowser.RenderProcessGone. To test it: killall -9 "exclr8cef_demo Helper (Renderer)" from Terminal while the page is loaded. The demo logs the termination cause to the event console then auto-reloads after 800 ms.

CEF init + request headers

The demo passes a CefSettings to Cef.SetInitSettings before init (cache, full UA, persistent cookies). It also subscribes to CefBrowser.ResourceRequest and injects an X-Exclr8Cef-Demo: hello header on every outgoing request. Click to round-trip through httpbin and verify both.

HTTP auth + Find-in-page

Click to load a URL guarded by HTTP-Basic auth — host pops the credentials dialog. The endpoint expects user / passwd; correct values render a JSON success. Cmd/Ctrl+F over the page opens the in-page find bar; ↑/↓ or Enter / Shift-Enter cycles matches.

JS → host bridge

Every renderer V8 context gets a window.exclr8cef.invoke(method, argsJson) function wired up automatically. Calling it sends a CefProcessMessage to the browser process, which fires CefBrowser.JsInvoke on the host. Fire-and-forget for v1 — no return value to JS yet (return-values + Promise interop are v2).
